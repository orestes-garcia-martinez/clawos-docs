## Trust is part of the product

ClawOS is not positioned as a loose assistant wrapper. Trust comes from architectural decisions.

## Key trust signals

- first-party skills only
- no runtime marketplace installs
- clear separation between platform and skill responsibilities
- billing source of truth outside the client
- minimal resume storage surface
- explicit session and identity boundaries

## What the docs should reinforce

The docs site should explain these choices plainly instead of hiding them behind vague marketing language.
